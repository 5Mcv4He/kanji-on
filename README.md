## 为什么不是「背」而是「推导」

作为中文母语者，你天然掌握了一项日本人没有的武器——**汉语发音**。

| 你已有的直觉 | 可以推导的音读 |
|:--|:--|
| 拼音声母 `j` | カ行（健·建·見·鍵·剣…） |
| 拼音韵母 `-ou` | ウ段長音（有·由·友·右·油…） |
| 声符「方」 | ホウ（坊·妨·房·放·肪…） |
| 中古音「清」韵 + ing | セイ（清·晴·静·精·情…） |

> 这不是联想记忆法。这是**历史语言学规律**——日语汉音源自唐代长安音，而现代汉语方言和中古音重构可以反向桥接这个对应关系。

- **系统精选**：贪婪集合覆盖压缩 107:1，字覆盖率 100%
- 每条规则标注置信度（确定 ≥80% / 大概率 60-80% / 有时 40-60% / 偶尔 <40%），你决定用哪些

## 五条铁律（看一眼就能用）

1. **声母定行** — 拼音声母 `j` → カ行、`zh` → サ行、`b` → ハ行 … （准确率 ~78%）
2. **鼻音定长** — 拼音韵尾 `-n/-ng` → 对应长音或拨音（准确率 98%+）
3. **声符定音** — 看到「方」→ 读ホウ，看到「包」→ 读ホウ（准确率 ~57-78%）
4. **入声定短** — 中古入声字 → 日语音读短促音节（ツ·ク·キ·チ·フ）
5. **呉漢分層** — 同一个汉字两种音读？呉音保留濁音、漢音清化 → カク/ガク 都说得通

> 五条铁律 ≈ 你中学语文知识的日语版投射。学完 N5 拼音对应关系，剩下的只是扩展声符和 MC 特征。

## 快速开始

```bash
# 安装
pip install pandas numpy xgboost scikit-learn openpyxl pypinyin

# 查询任意 JLPT 汉字的完整音训分析
python deep_lookup.py 鬱
python deep_lookup.py 学 --json    # JSON 输出
```

规则已预生成在 `output/` 目录，开箱即用。无需数据库。

## 项目全景

```
三重桥接 ──→ 穷尽特征枚举 ──→ 规则发现 ──→ 置信度标注 ──→ 贪婪精选 ──→ 输出
    │            │                │              │              │
    │    拼音特征 (声母/韵母/声调/鼻尾)         │         XGBoost 蒸馏
    │    中古音特征 (声母/韵母/等/開合/清濁/入声)  │         Tier 1~5 分級
    │    部件特征 (声符/部首/非部首/画数)         │         4档置信度
    │                                          │
    └────────────── 46,848 汉字的 MC 映射表 ──→ 呉音/漢音 推导引擎
```

**ML 实验线**：统计基线 → 神经网络 (M0-M3) → 改进版 (residual + label smoothing) → XGBoost → 决策树蒸馏为 if-then 规则。模型代码在 `models/`。

**Web 应用**：`web/index.html` 提供交互式汉字查询，支持 N5-N1 切换和中古音特征筛选。`python build_web_data.py` 生成数据文件。
