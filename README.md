# Kanji-On

> 日语汉字音读规律 · 穷尽规则体系
>
> 面向中国 JLPT 学习者的汉字音读规则发现系统。以现代汉语拼音、中古音（廣韻）、汉字部件为三重桥接，穷尽发现所有人类可理解的「特征 → 日语音读」对应规则。每一条规则标注 JLPT 层级、难度和置信度。

---

## 为什么你需要这个

作为中国 JLPT 学习者，你天然掌握了一项日本人没有的武器——**汉语拼音**。现代汉语拼音、中古音（廣韻）特征与日语音读之间存在系统性对应规律。这个项目把这些规律**穷尽地**找了出来。

不是「记住这个汉字读什么」，而是「**理解为什么这样读，然后举一反三**」。

| 你已有的知识 | 可以推导的音读 |
|-------------|-------------|
| 拼音 `j+ian4` | ケン（健·建·見·鍵·剣...） |
| 拼音 `y+ou2` | ユウ（有·由·友·右·油·遊...） |
| 声符「方」 | ホウ（坊·妨·房·放·肪·紡...） |
| MC韵母「清」+ 拼音韵母 `ing` | セイ（清·晴·静·精·情·請...） |

## 核心设计

### 三重桥接

```
汉语拼音 ──→ 日语音读    中国小学就掌握的知识
中古音（廣韻）──→ 日语音读   更细粒度的音位映射，支撑呉漢分離
汉字声符/部件 ──→ 日语音读   看到这个部件就读这个音
```

### 穷尽发现 × 四档置信度

不强过滤低精度规则。每条规则标注置信度，用户自行选择使用：

| 置信度 | 精度 | 含义 | N1 数量 |
|--------|------|------|---------|
| **确定** | ≥ 80% | 语言学规律，强烈推荐记忆 | 4,218 |
| **大概率** | 60-80% | 多数情况适用，注意例外 | 3,864 |
| **有时** | 40-60% | 可参考，反例较多 | 5,065 |
| **偶尔** | < 40% | 弱参考，仅穷尽检索时保留 | 66,685 |

### 五级难度分层

| Tier | 难度 | 特征类型 | 示例 |
|------|------|---------|------|
| ★ | 最易 | 单拼音 / 单 MC 特征 | `拼音 y+ou → ユウ` |
| ★★ | 容易 | 部件 / 结构 / 声符 | `声符「包」→ ホウ` |
| ★★★ | 中等 | 双特征配对（28 种 MC 内部 + 32 种跨域） | `MC韵摄「止」+ MC声类「齒」→ シ` |
| ★★★★ | 较难 | XGBoost 复合挖掘 | `MC韵母=清 AND 鼻音=-ng → セイ` |
| ★★★★★ | 最难 | 三层特征 / 决策树蒸馏 | `MC声类=齒 AND MC韵母=仙 AND 非入声 → セン` |

### 三条渐进学习路径

| 路径 | Tier 范围 | N5 规则数 | N5 覆盖 | N1 规则数 | N1 覆盖 | 适合 |
|------|----------|----------|---------|----------|---------|------|
| **A·快速入门** | 1-2 | 156 | 38% | 321 | 36% | 只用拼音和声符知识 |
| **B·深度学习** | 1-3 | 197 | 47% | 418 | 44% | 加入 MC 双特征配对 |
| **C·追求极致** | 1-5 | 334 | 96% | 746 | 85% | 含中古音推导 + 呉漢分離 |

### 呉音・漢音分離

基于 MC → 日语音系推导引擎（`predict_go_kan.py`），当 MC 特征在两个历史层次产生不同日语音读时，规则标注 `reading_type`（go-on / kan-on / general）。

| MC 特征 | 呉音反射 | 漢音反射 |
|---------|---------|---------|
| 全濁声母 (b/d/g/dz) | 保留濁音（バ/ダ/ガ/ザ行） | 变清音（ハ/タ/カ/サ行） |
| 明母 (m-) | マ行 | バ行 |
| 日母 (ny-) | ナ/ニャ行 | ザ/ジャ行 |
| 梗攝 + 非入声 | -ャウ | -エイ |

## 使用方式

```bash
# 完整的音訓规则分析 + 词例 + 学习建议
python deep_lookup.py 鬱

# JSON 输出（供程序调用）
python deep_lookup.py 鬱 --json

# 快速查询
python kanji_lookup.py 学 --level N5

# 批量查询
python kanji_lookup.py 日本語 --level N3
```

输出包含：基本信息（拼音/部首/声符/画数/意义）→ 中古音完整分解（所有 MC 读法 + 呉漢标注）→ 每条音读的匹配规则 → 训读语义群分析 → 学习路径建议。

## 项目结构

```
kanji-on/
├── tiered_rules.py              # ★ 核心：穷尽规则生成 + 贪婪精选
├── deep_lookup.py               # ★ 深度汉字分析（主 CLI 入口）
├── kanji_lookup.py              # 快速汉字查询
├── build_level_dataset.py       # N5-N1 分层数据集构建
├── build_dataset.py             # 全量数据集构建
├── build_mc_data.py             # 廣韻 → 中古音映射表构建
├── enrich_mc_features.py        # 数据集 MC 特征增强 + 呉漢标注
├── predict_go_kan.py            # MC → 呉音/漢音 音系推导引擎
├── extract_onkun_rules.py       # 音訓判別规则
├── train_level_xgboost.py       # XGBoost 多分类器训练
├── distill_xgboost_rules.py     # 决策树蒸馏（XGBoost → if-then）
├── merge_cross_level_rules.py   # 跨层规则一致性分析
├── dataset/                     # N5-N1 累计数据集 (CSV)
├── output/                      # 规则输出 (JSON + MD)
├── models/saved/                # 预训练 XGBoost 模型 + 编码器
└── docs/                        # 项目报告 + 规格书
```

## 安装与使用

### 环境

```bash
pip install pandas numpy xgboost scikit-learn openpyxl pypinyin
```

无需数据库，数据源文件已在仓库中。

### 规则已预生成

所有层级的规则 JSON 和 MD 报告在 `output/` 目录下，无需重新生成即可查询任意 JLPT 汉字：

```bash
python deep_lookup.py 鬱
```

### 重新生成规则

```bash
# 单级穷尽规则 + 精选 + MD 报告
python tiered_rules.py N5

# 输出:
#   output/n5_tiered_rules_exhaustive.json   (全部 26,377 条)
#   output/n5_tiered_rules.json              (精选 334 条)
#   output/n5_tiered_rules.md                (可读报告)
```

### 重新训练模型（可选）

```bash
python train_level_xgboost.py N1        # XGBoost 训练
python distill_xgboost_rules.py N1      # 决策树蒸馏
```
