## 1. 项目概览

### 1.1 一句话定义

这是一个**把现代汉语拼音、中古音（廣韻）音系特征、汉字部件结构、以及机器学习挖掘四者组合起来，穷尽发现所有人类可理解的「拼音/中古音 → 日语音读」对应规则**的系统。每一条规则都标注了 JLPT 层级（N5→N1）、难度层级（★〜★★★★★）和置信度（确定/大概率/有时/偶尔）。

### 1.2 规模一览

| 指标 | N5 | N4 | N3 | N2 | N1 |
|------|-----|-----|-----|-----|-----|
| 汉字数 | 653 | 1,074 | 1,512 | 1,875 | 2,179 |
| 音读类别 | 127 | 170 | 206 | 218 | 226 |
| 穷尽规则数 | 26,377 | 42,249 | 58,179 | 70,958 | 79,832 |
| 精选规则数 | 334 | 470 | 568 | 665 | 746 |
| 确定规则(≥80%) | 938 | 1,917 | 2,825 | 3,511 | 4,218 |
| 覆盖率 | 100% | 100% | 100% | 100% | 100% |

### 1.3 技术栈

- **Python 3.14** + pandas / numpy
- **XGBoost** 多分类器（训练/蒸馏/挖掘）
- **廣韻 rhyme_book.csv**（qieyun 库）→ 中古音特征
- **pypinyin** → 现代汉语拼音特征
- **漢字検索V2.xlsm** (K'sBookshelf 汉字林, 46,848字) → 部首/声符/画数
- **紅宝書 word.xlsx** (华东理工大学出版社, ~9,500 JLPT词汇) → 层级映射

## 2. 问题定义

### 2.1 核心问题

**中国 JLPT 学习者看到一个新汉字时，怎么知道它的日语音读？**

现有方法的问题：
- **死记硬背**：每个汉字单独记音读，2,000+ 汉字 = 2,000+ 次记忆负担
- **日本人直觉**：日本教材默认母语者已内化音读规律，不给规则
- **传统音韵学**：规则精确但门槛高，需要懂中古音三十六字母、二百零六韵

### 2.2 为什么中国学习者有独特优势

现代汉语拼音和中古音（廣韻）与日语音读之间存在**系统性对应规律**——这是日语从古汉语借入汉字读音时留下的历史痕迹。

**三重桥接**：
1. **拼音 → 音读**：声母、韵母、声调——中国小学就掌握的知识
2. **中古音 → 音读**：廣韻的声母/韵母/等/開合/清濁——把拼音映射细化
3. **声符 → 音读**：汉字本身的形声字结构——"看到这个部件就读这个音"

### 2.3 穷尽发现的含义

不是"找几个最强的规则就够了"，而是：

> **只要是人类语言可以理解的规律，全部找出来。无论什么方法。**
> 学习者可以选择性地使用，但系统负责穷尽。

这意味着：
- 不强过滤低精度规则（标注置信度即可）
- 不限于单一特征类型（拼音、中古音、部件、部首、画数全部纳入）
- 不限于单一发现方法（统计计数 + 穷尽配对 + XGBoost挖掘 + 决策树蒸馏）

## 3. 设计哲学

### 3.1 五条核心原则

| 原则 | 含义 |
|------|------|
| **穷尽优先于精准** | 先全部发现，再标注置信度，不预先砍掉弱规则 |
| **分层优先于扁平** | 每条规则同时标注 JLPT 层级和 5 级难度，支持渐进学习 |
| **可解释优先于黑箱** | 所有规则的格式是"如果（人类可理解的条件）→ 那么（音读）"，不用模型概率直接做规则 |
| **显式生成优先于隐式挖掘** | 28 种 MC 内部配对 + 32 种跨域配对全部显式生成，XGBoost 挖掘只用于填补剩余 |
| **双重输出** | 穷尽版（所有规则，用于检索） + 精选版（贪婪集合覆盖，用于渐进学习） |

### 3.2 用户画像

- **N5-N4 初学者**（~650-1,000 汉字）：用基础的拼音桥接 + 部件规则，不需要懂中古音
- **N3-N2 中级者**（~1,500-1,900 汉字）：加入中古音单特征规则，理解声母/韵母/等/清濁
- **N1 高级者**（~2,200 汉字）：使用全部规则类型，包括跨域配对和 Go/Kan 分离规则

## 5. 数据层

### 5.1 数据集结构

每个 `dataset/{level}_dataset.csv` 包含 **34+ 列**，分为四大类：

**基础属性（8列）**：kanji, pinyin, pinyin_initial, pinyin_final, pinyin_tone_num, onyomi, kunyomi, meaning

**结构特征（6列）**：radical, non_radical_part, total_strokes, radical_strokes, non_radical_strokes, nasal_coda

**中古音特征（14列）**：
mc_initial, mc_initial_cat, mc_voicing, mc_rhyme, mc_rhyme_cat, mc_grade, mc_openness, mc_tone, mc_is_entering, mc_entering_coda, mc_readings_json, mc_reading_count, mc_reading_source, mc_reading_types

**呉漢特征（3列）**：mc_goon_primary, mc_kanon_primary, mc_divergent

### 5.2 累计层级策略

N5 ⊂ N4 ⊂ N3 ⊂ N2 ⊂ N1

每个高层级包含所有低层级的汉字 + 该层级新增的汉字。这让跨层规则可以追踪：一条在 N5 发现的规则，到 N1 是否仍然有效？

**关键发现**：0 条规则跨层退化（从 N5 到 N1 精度下降 >15%）。这意味着早期发现的规则是稳定的，不会因为看到更多汉字而失效。

## 8. 置信度标注

### 8.1 四档标注（替代硬过滤）

```
确定   (≥80%): 语言学规律，强烈推荐记忆
大概率 (60-80%): 多数情况适用，注意例外
有时   (40-60%): 可参考，但反例较多
偶尔   (<40%): 弱参考，仅穷尽检索时使用
```

**为什么不砍掉低精度规则？**

例如 `MC韵摄「臻」→ シン` 只有 10% 精度（171 字中仅 17 字正确），但这句话本身是一条**正确的语言学知识**——臻摄字确实可能读 シン。低精度反映的是"一对多映射"（一个 MC 特征对应多个日语音读），不是规则错误。保留这类规则可以让学习者在 deep_lookup 中看到"为什么这个字也可能读别的音"。

### 8.2 跨级分布

| 置信度 | N5 | N4 | N3 | N2 | N1 |
|--------|-----|-----|-----|-----|-----|
| 确定 | 938 | 1,917 | 2,825 | 3,511 | **4,218** |
| 大概率 | 1,373 | 2,217 | 2,700 | 3,521 | 3,864 |
| 有时 | 2,068 | 3,034 | 3,911 | 4,759 | 5,065 |
| 偶尔 | 21,998 | 35,081 | 48,743 | 59,167 | 66,685 |

**确定规则从 N5 到 N1 增长 4.5 倍**（938 → 4,218）——更高的层级有更多的汉字来确认规律。

## 9. 呉音・漢音分離

### 9.1 背景

日语音读有两个主要历史层次：
- **呉音（5-6世纪）**：经由朝鲜半岛（百济）传入，更古老
- **漢音（7-8世纪）**：经由遣唐使传入，后成为标准

当 MC 特征在两个层次产生**不同**的日语音读时，一个汉字可能有两个完全不同的音读。例如「行」：呉音ギョウ、漢音コウ。

### 9.2 实现方式

由于 KANJIDIC2 的 `on_type` 属性在实际 XML 数据中为空白（DTD 定义了但数据未填充），系统采用**音系推导引擎** (`predict_go_kan.py`) 来推断 reading_type：

**关键音系对比**：

| MC 特征 | 呉音反射 | 漢音反射 | 信号 |
|---------|---------|---------|------|
| 全濁声母 (b/d/g/dz 等) | 保留濁音(バ/ダ/ガ/ザ行) | 变清音(ハ/タ/カ/サ行) | 最强 |
| 明母 (m-) | マ行 | バ行 | 强 |
| 日母 (ny-) | ナ/ニャ行 | ザ/ジャ行 | 强 |
| 泥/娘母 (n-/nr-) | ナ行 | ダ行 | 强 |
| 梗攝 + 非入声 | -ャウ | -エイ | 中 |

### 9.3 产出

每条 MC 内部配对规则标注 `reading_type`：
- `general`（78,302 条）：呉漢反映相同，不需要区分
- `divergent`（1,530 条）：呉漢反映不同，该规则只适用于特定历史层次

divergent 规则示例：
- `MC声母「並」+MC韵母「模」→ ホ` — 全濁声母的呉音/漢音不同反射
- `MC声母「並」+MC韵母「陽」→ ホウ` — 同样涉及呉漢分离

### 9.4 数据集标注

每个汉字通过 `enrich_mc_features.py` 标注：
- `mc_divergent`（1 = 该汉字有呉漢異读）
- `mc_goon_primary`（JSON：主要呉音对应的 MC 特征）
- `mc_kanon_primary`（JSON：主要漢音对应的 MC 特征）

N1 数据集中 1,530 条规则标记为 divergent。

## 10. 中古音桥接

### 10.1 为什么需要中古音

拼音桥接直接有效（`j+ian4 → ケン` 精度 100%），但有一类问题拼音解决不了：

- 「行」为什么有时读コウ、有时读ギョウ？—— 拼音都是 x+ing2
- 「大」为什么有时读ダイ、有时读タイ？—— 拼音都是 d+a4

答案在中古音里：
- 「行」MC 匣母庚開二平 (ɦ-æŋ) → 呉音ギョウ / 漢音コウ
- 「大」MC 定母泰開一去 (d-ɑi) → 呉音ダイ / 漢音タイ

中古音提供了拼音不具备的**清濁**和**等**信息，这是区分呉漢的关键。

### 10.2 数据来源

廣韻（1008年）是现存最完整的中古音韵书。通过 `qieyun` Python 库的 `rhyme_book.csv`（包含 18,212 条记录），解析 `最簡描述` 字段（格式如「從開三支平」）提取中古音特征。

### 10.3 中古音特征在规则中的角色

| MC 特征 | 语言学含义 | 对日语音读的影响 |
|---------|----------|----------------|
| 声母 (initial) | 辅音类型 | 决定日语音读的辅音行 (カ/ガ/サ/ザ 等) |
| 韵母 (rhyme) | 主元音+韵尾 | 决定日语音读的元音段 |
| 等 (grade) | 介音类型 | 影响拗音 (ャ/ュ/ョ) 有无 |
| 開合 (openness) | 合口介音 | 影响是否出现 ワ 行音 |
| 声调 (tone) | 平/上/去/入 | 入声对应短促音节(ク/ツ/フ)，其他影响元音长短 |
| 清濁 (voicing) | 声母清浊 | 决定呉漢差异（全濁→呉濁/漢清） |
| 韵摄 (rhyme category) | 16 大类韵母 | 粗粒度的韵→音读映射 |
| 入声韵尾 | -p/-t/-k | 决定日语音读末音 (フ/ツ/ク) |

## 11. 规则挖掘方法

### 11.1 三种发现策略

#### A. 统计直接计数（Tier 1-3）

对所有单特征和显式定义的配对，直接分组计数：P(音読 | 特征值) = 该特征值下该音読的数量 / 该特征值的总出现次数。

**优点**：100% 可解释，无需模型
**覆盖**：Tier 1-3 的 43,910 条规则（N1）

#### B. XGBoost 挖掘（Tier 4-5）

1. 训练 XGBoost 多分类器（max_depth=8, 300 estimators）
2. 对 XGBoost 正确预测的汉字，提取其特征值（如 `鼻音=-ng`, `声母=t`, `MC韵母=清`）
3. 在这些 XGBoost-correct 汉字的特征值集合中，挖掘高频特征值对和三元的共现模式
4. 自适应阈值：`max(2, len(indices) * 0.35)` — 覆盖 ≥35% 的正例子集的特征值才进入挖掘

**优点**：发现人类显式枚举遗漏的复合模式
**覆盖**：Tier 4-5 的 35,922 条规则（N1）

#### C. 决策树蒸馏（Tier 5 补充）

对每个频次 ≥5 的音读训练 OvR 决策树代理模型（max_depth=5, min_samples_leaf=4），从 XGBoost 的判别行为中提取 if-then 路径。

**优点**：规则来自 XGBoost 的"判断逻辑"而非原始数据，能捕获非线性交互
**覆盖**：84 条蒸馏规则（N1）

### 11.2 贪婪集合覆盖（精选版）

从穷尽规则集中用**贪婪集合覆盖**选出最小规则集：

```
score = len(newly_covered_kanji) × accuracy²
```

`accuracy²` 权重确保高分规则优先——一条 95% 精度的规则比 50% 精度的规则有 **3.6 倍的优先级**。

**精选效果**：79,832 → 746 条规则（N1），压缩比 107:1，仍保持 100% 覆盖率。

### 12.4 关键洞察

1. **Tier 1-2 即可覆盖 30-37%**：单特征规则（拼音+声符）是最具性价比的学习起点

4. **跨层零退化**：N5 发现的规则到 N1 精度不低于 70%，说明规则质量稳定

## 14. 边界与局限

### 14.1 范围边界

| 在范围内 | 不在范围内 |
|---------|----------|
| JLPT N5-N1 音读预测 | 非 JLPT 汉字（可用 N1+/N1++ 扩展） |
| 音读 (on'yomi) 规则 | 训读 (kun'yomi) 规则生成（有独立的 kun 语义分析模块，但未做穷尽） |
| 呉音/漢音 推断（音系推导） | 唐宋音/慣用音 区分 |
| 现代汉语拼音 + 中古音 + 结构特征 | 上古音、吴音以外的方言层 |
| 单汉字→音读的映射 | 连濁 (rendaku)、音便 (onbin) 等词汇层音变 |
| 形声字（占 80%+ 汉字） | 会意字/指事字/象形字（本身无规则可寻，只能死记） |

### 14.2 精度边界

- **音读预测不是 100% 精确**：即使最好的单特征规则（如 `j+ian4 → ケン` 100%），其覆盖范围只有 4 个汉字。高精度和高覆盖之间存在取舍。
- **一对多映射**：一个 MC 特征对应多个日语音读是常态（如臻攝→シン/ジン/イン等），低精度规则标注为"偶尔"保留供参考。
- **呉漢推断基于音系推导而非标注数据**：KANJIDIC2 的 on_type 字段为空，预测可能对 20-30% 的边缘情况不准确。

### 14.3 数据边界

- **粤语/闽南语/客家话缺失**：当前系统只用普通话拼音和中古音，没有加入保留更多中古音特征的现代汉语方言。
- **KANJIDIC2 完整标注缺失**：呉音/漢音/唐宋音/慣用音的真实标签在实际 XML 数据中未填充，系统用规则推断替代。
- **部首/声符数据仅覆盖 V1+V2 中的汉字**（~47K），极罕见汉字可能无结构数据。

### 14.4 使用边界

- **这不是一个「输入拼音→输出正确音读」的预测系统**。它是一个**规则知识库**。用户需要通过规则理解规律，而不是期待系统给出唯一正确答案。
- **精选规则集的 100% 覆盖率是指「至少有一条规则可以解释」，不是「第一条规则就是正确的」**。学习者需要通过多条规则 + 上下文（单词/词例）来判断。

### 14.5 已知未解决的问题

1. **声符歧义**：同一个声符可能对应多个音读（如「寺」→ジ/タイ/シ等），系统目前报告所有可能性但不做消歧
2. **入声韵尾与日本音读末音的不完全对应**：-p→フ的演变在历史上经过 -p→-ɸ→-u，部分汉字末音难以从 MC 直接推导
3. **極低頻汉字**（<3 次出现）：无法生成有统计意义的规则，在穷尽集中被 min_support=2/3 过滤
