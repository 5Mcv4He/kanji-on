# 日语汉字读音规律发现系统 — 项目规格书

## 1. 项目概述

基于日语汉字音训对照表和 JLPT 分级词汇表，利用可解释 AI 模型，针对不同学习层次（N5–N1++）自动发掘用于推测日语汉字读音的规律。规律需具备可解释性（人类可理解），并综合输出。

**设计原则**：准确率在「人类可理解」的约束下推到极限。黑箱模型仅作为模式发现引擎，最终产物是可学习的规则，不是模型权重。

**核心假设**：现代汉语拼音与日语音读之间存在系统性对应规律。声母、韵母、声调、鼻音韵尾——中国小学就掌握的知识——是预测日语汉字读音的最强信号。

## 3. 学习层级定义

共 7 层。每层包含前一层的汉字集合（累积制）。

| 层 | 汉字范围 | 单词范围 | 音训读数据源 |
|----|---------|---------|------------|
| **N5** | N5单词中出现的全部汉字（约653字） | 仅N5单词（约934词） | V2优先、V1补充 |
| **N4** | N5+N4单词中出现的全部汉字（约1,074字） | N5+N4单词（约1,999词） | V2优先、V1补充 |
| **N3** | N5+N4+N3单词中出现的全部汉字（约1,512字） | 累计约4,006词 | V2优先、V1补充 |
| **N2** | N5–N2单词中出现的全部汉字（约1,875字） | 累计约6,250词 | V2优先、V1补充 |
| **N1** | N5–N1单词中出现的全部汉字（约2,179字） | 累计约9,172词 | V2优先、V1补充 |
| **N1+** | V1 全部收录汉字（6,354字） | N1为止的全部单词 | V2优先（6,353字重叠）、V1补充 |
| **N1++** | V2 全部收录汉字（46,848字） | N1为止的全部单词 | V2 |

### 3.1 「基准答案」的定义（重要）

各层中汉字的「正确读法」，**不是 V1/V2 中标注的全部音训读，而是该层单词表中实际出现的读法**。

例如：N5层中的「上」，在 N5 单词中以 `あ（上げる）·うえ（上）·うわ（上着）·ジョウ（上手）·のぼ（上る）` 出现。V2 中还标注了 `ショウ·かみ`，但它们在 N5 单词中没有出现，因此不计入 N5 层的评价范围。

从单词表提取汉字读法的方法：
1. 单汉字词（如 `上→うえ`）→ 假名即为读法
2. 带送假名詞（如 `上げる→あげる`）→ 从假名中扣除送假名对应部分，剩余为汉字读法
3. 复合词（如 `上手→じょうず`）→ 通过汉字与假名的对齐提取各汉字的读法

V1/V2 的角色：
- **特征提供**：声符（非部首部）、部首、画数、完整音训列表
- **参考显示**：查字工具中显示「其他层级会出现的读法」
- **不作为评价基准**：V1/V2 的完整读法列表不作为该层的正确答案

## 4. 可解释 AI 模型

### 4.1 模型选定方针

同时使用多种可解释性层级的模型，层级式地发现规律：

| 优先级 | 模型 | 可解释性 | 作用 |
|--------|------|---------|------|
| 1 | **统计模型**（部件计数投票） | 100%（规则本身） | 发现100%确定规则和强规则 |
| 2 | **拼音桥接**（声母+韵母→音读映射） | 100% | 面向中国学习者最强的预测器 |
| 3 | **决策树 / XGBoost** | 高（特征重要性、树结构可视化、SHAP分析） | 发现非线性交互。哪个特征在什么条件下起作用 |
| 4 | **神经网络集成**（现有 BetterModel 等） | 低（黑箱） | 覆盖上述方法无法处理的残余案例，发现规则化候选 |

### 4.2 规则形式

以下列形式表达规则。所有规则需附带「分层正确率」「适用汉字数」「例外列表」：

1. **拼音桥接规则**：`g + uan → カン`（100%, 23字）。声母+韵母组合直接决定音读。
2. **声符规则**：`冓 → コウ`（100%, 9字）。非部首部（声符）决定音读。
3. **部首语义域规则**：部首所示语义范畴与训读的对应（如 `言`部首→语言·告知相关训读）。
4. **同训异字规则**：同一训读中不同汉字的语义分工（如 `とる`→取·採·撮·捕·執·穫）。
5. **结构规则**：部首画数·非部首部画数之比等结构特征对应的读法倾向。
6. **复合规则**：上述规则的 AND/OR 组合（如「声符X + 部首Y → 必读Z」）。

### 4.3 规则评价基准

按规则类型分别设定最低正确率阈值：

| 规则类型 | 最低正确率 | 理由 |
|---------|-----------|------|
| 拼音桥接（声母+韵母） | 80% | 体系性强。低质量规则会导致错误学习 |
| 拼音桥接（仅韵母） | 70% | 信息量较少，适当降低阈值 |
| 声符规则（确定） | 100% | 仅将「无例外」作为确定规则 |
| 声符规则（强） | 80% | 明确标注包含1-2个例外 |
| 声符规则（弱） | 60% | 参考信息。附信頼度提供 |
| 结构规则 | 不设阈值（参考） | 仅作为辅助信息提供 |

## 5. 输出功能一：分层规律汇总文件

针对各层（N5, N4, N3, N2, N1, N1+, N1++）生成以下文件：

### 5.1 分层规则文件格式

```csv
层, 规则ID, 规则类型, 规则描述, 正确率, 适用汉字数, 适用汉字列表, 例外汉字列表, 拼音（如适用）, 首次出现层
```

### 5.2 规则降级追踪

同一规则在不同层级的正确率变化追踪：

```
N5: g+uan→カン  100% (5/5)  [官観管関館]
N4: g+uan→カン  100% (8/8)  [冠巻貫 追加]
N3: g+uan→カン   92% (11/12) [棺 追加, 権 例外]
```

这种追踪本身对学习者有参考价值——「N3才首次出现例外」，意味着可以在 N5-N4 放心使用这条规则。

### 5.3 分层摘要

```
# N5 规则摘要
- 总规则数: XX
- 确定规则（100%）: XX
- 强规则（≥80%）: XX
- 弱规则（≥60%）: XX
- 规则覆盖率: XX%（N5汉字中至少有一条规则覆盖的比例）
- Top-1 正确率（仅规则）: XX%
- Top-3 正确率（仅规则）: XX%
```

## 6. 输出功能二：单字规律与信息查询

### 6.1 输入

```
python kanji_lookup.py <汉字> [--level N3]
```

`--level` 指定时，在该层累计范围内评价。未指定时全层。

### 6.2 输出内容

1. **基本信息**
   - 汉字、音读（该层有效读法）、训读、拼音、部首、画数、JLPT等级、意思

2. **规则分析**
   - 适用的全部规则（按规则类别）
   - 每条规则在该层的正确率
   - 每条规则的「兄弟字」（同规则适用的其他汉字）
   - 每条规则的「例外字」（同条件但不适用规则的汉字）

3. **AI 预测**
   - XGBoost + 统计模型 + 拼音桥接 + 神经网络集成给出的 Top-5 预测
   - 各候补的信頼度
   - 预测依据说明（哪些特征贡献了：SHAP值）

4. **同音同训汉字**
   - 该层内具有相同音读的汉字一览
   - 该层内具有相同训读的汉字一览

5. **包含单词一览**
   - 该层中包含此汉字的全部单词
   - 各单词的汉字表记·假名表记
   - 各单词中此汉字的读法

6. **训读分析**
   - 部首的语义域
   - 存在同训异字组时的详细分析（核心义·默认汉字·各汉字语感差异·典型例句）

---

## 7. 输出功能三：规则验证演示数据

### 7.1 分层验证报告

对各层生成以下内容：

1. **规则基线对比**
   - 随机猜测的正确率（基线）
   - 仅规则预测的正确率（Top-1 / Top-3 / Top-5）
   - 规则 + XGBoost 的正确率
   - 全模型集成的正确率

2. **规则覆盖度分析**
   - 规则覆盖的汉字比例
   - 未覆盖汉字的特征（无声符字、多音字等）
   - 覆盖率的分层变化

3. **错误分析**
   - 多音歧义导致的错误（预测了另一个有效读法）
   - 无声符导致的错误（象形字等）
   - 拼音例外（偏离同拼音组主流）
   - 声符信頼性不足（统计上规律性弱）

4. **样本演示**
   - 各层选取代表性汉字
   - 规则预测结果与实际读法的对比表
   - 可视化：热力图·混淆矩阵·特征重要度图

### 7.2 验证数据文件格式

```csv
层, 汉字, 正确音读, 规则预测Top-1, 一致, 预测Top-3, Top-3包含否, 适用规则, 错误类别
```

## 8. 实现计划

### Phase 1: 数据集重建（从 N5 开始）

1. 从 word.xlsx 解析 MP3 文件路径，提取全部单词的 N5-N1 等级
2. 从各层单词集合中提取汉字，对齐单词内的读法
3. 从 V2 合并各汉字的结构信息（非部首部·部首CD·画数等）
4. 通过 pypinyin 附加拼音·声母·韵母·声调·鼻音韵尾
5. 输出各层数据集 CSV

### Phase 2: N5层统计分析·规则提取

1. 基于非部首部的部件计数投票（从V1「全部件」→V2「非部首部」，精度有望提升）
2. 拼音桥接（声母+韵母→音读）的构建
3. 决策树探索非线性规则
4. 输出 N5 专用规则集

### Phase 3: N5→N1 渐进训练

逐层提升，反复进行规则的追加·降级·精炼。

### Phase 4: N1+·N1++ 扩展

从 V1 全量→V2 全量扩展。对生僻字标注规则稳定性注意事项。

### Phase 5: 工具·报告生成

kanji_lookup.py 重构（对应 XGBoost + 新规则引擎），验证报告自动生成。
