# Kanji-On

> 中国語話者のための漢字音読規則の網羅的体系
>
> 現代中国語（拼音）、中古音（廣韻）、漢字構成要素の3つの橋渡しを通じて、「特徴 → 日本語音読」の対応規則を網羅的に発見。全規則にJLPTレベル・難易度・信頼度を付与。「読み方を覚える」のではなく「**なぜそう読むのかを理解する**」。

[中文](README.md) | [English](README.en.md)

## コア設計

### 3つの橋渡し

```
拼音 ──→ 日本語音読        中国人なら小学校で習得済み
中古音（廣韻）──→ 日本語音読   より精密な音素マッピング、呉漢分離を支える
漢字の声符/部件 ──→ 日本語音読   この部件を見たらこの音
```

### 5つの鉄則 vs システム精選規則 — 実測比較

紅宝書 N5-N1 全語彙での実測：

| 手法 | 規則数 | N5 カバー | N1 カバー | 備考 |
|------|--------|----------|----------|------|
| 沪江2規則（声母→行 + 鼻音→長音） | 2 | 85.8% | 85.3% | ネットでよく見る方法 |
| **5つの鉄則**（+ 部件→音 + 入声 + 呉漢） | 5 | 96.0% | 91.4% | 本プロジェクトの最簡規則 |
| システム精選（Tier 1-3、高精度） | 94-298 | 62% | 84% | 精度 98% 以上 |
| システム精選（全5 Tiers） | 334-746 | **100%** | **100%** | 網羅的 |

> 5つの鉄則は沪江より 6-11% 多くカバー、たった 3 規則追加するだけ。**推奨**：まず鉄則で高速入門（91-96%）、その後 Tier を段階的に追加。

詳細: [`output/iron_laws_analysis.json`](output/iron_laws_analysis.json)

### 音訓判別 — 単語を見て、読み方をどう判断するか

紅宝書全語彙から抽出した 3 つの形式規則：

| 規則 | 説明 | 実測精度 |
|------|------|----------|
| 送仮名則 | 漢字+送仮名 → 訓讀（食べる·見る） | **99%+** |
| 連続漢字則 | 漢字2字以上連続 → 音讀（学生·電車） | **98%+** |
| 単漢字優先則 | 漢字1字 → まず訓讀、なければ音讀 | ヒューリスティック |

例外：熟字訓（今日→きょう）、重箱読み（場所→ばしょ）、湯桶読み（夕刊→ゆうかん）。

| パス | Tier | N5 カバー | N1 カバー | 対象者 |

### 呉音・漢音分離

MC → 日本語音韻導出エンジン（`predict_go_kan.py`）に基づく：

| MC 特徴 | 呉音 | 漢音 |
|---------|------|------|
| 全濁声母 (b/d/g/dz) | 濁音保持（バ/ダ/ガ/ザ行） | 清音化（ハ/タ/カ/サ行） |
| 明母 (m-) | マ行 | バ行 |
| 日母 (ny-) | ナ/ニャ行 | ザ/ジャ行 |
| 梗攝 + 非入声 | -ャウ | -エイ |

## ML 実験

規則発見に加え、完全な ML 実験パイプラインを含む：

| モデル | 手法 |
|--------|------|
| ベースライン（統計） | P(on \| component) + Laplace 平滑化 |
| M0-M3（ニューラル） | 部件埋込 + 拼音/部首/画数特徴 |
| 改良版（residual） | 深層 MLP + Label Smoothing + AdamW |
| XGBoost → 蒸留 | 多クラス分類 → 決定木 → if-then 規則 |

モデル: `models/`。事前学習済み: `models/saved/`。蒸留規則: `output/n*_xgboost_rules.json`。
